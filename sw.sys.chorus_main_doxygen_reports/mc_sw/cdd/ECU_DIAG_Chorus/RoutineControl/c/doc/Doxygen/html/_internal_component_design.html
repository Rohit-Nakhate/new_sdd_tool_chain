Internal Component Design

Internal Component Design

The component is splitted into the follwing SW units:

-GWRoutines_EthDiag - GWRoutines_EthDiag process the start, stop and request result request for Ethernet.
-GWRoutines_Util - GWRoutines_Util sets the mode for the Ethernet physical Test.
-GWRoutines - GWRoutines process start, stop and request result requests and provides the relevant results.
-RoutineControl - RoutineControl provides the status information of the routine which includes, "was routine executed", "is routine finished","is routine running".
It also used to set the Notification status of the started,stopped,aborted and finished routines.

Requirements Allocation

RoutineControl component covers requirement present in the module 5.SWR_ICAS1_BSW and 5.1.1.SWR-L3_CRoute_Diagnostic. The requirements are present in chapter 2.18.7(Routines) of 5.SWR_ICAS1_BSW and chapter 3.4.2.10(RoutineControl) of 5.1.1.SWR-L3_CRoute_Diagnostic.

Software Unit Interfaces

Routine Control component provides the interfaces which are used by other SW Units.

RoutineControl SW unit provides the following interfaces which are used by GWRoutines_EthDiag Sw Unit.
-RC_WasRoutineExecuted :Called to check if requested routine was executed.
-RC_IsRoutineRunning: Called to check if requested routine is running.
-RC_StartRoutine: Called to start requested Routine.
-RC_StopRoutine:Called to stop requested Routine.
-RC_AbortRoutine:Called to abort requested Routine.
-RC_GetRoutineInfoByte:Called to get requested Routine result.
-RC_GetRoutineInfoOutReqStatus:Called to get requested Routine result status.

RoutineControl SW unit provides the following interfaces which are used by GWRoutines Sw Unit.
-RC_GetRoutineInfoOutReqStatus:Called to get requested Routine result status.
-RC_GetRoutineInfoByte:Called to get requested Routine result.
-RC_AbortRoutine:Called to abort requested Routine.
-RC_FinishRoutine::Called to finish the current Routine.
-RC_StopRoutine:Called to stop requested Routine.
-RC_StartRoutine: Called to start requested Routine.
-RC_IsRoutineRunning: Called to check if requested routine is running.
-RC_WasRoutineExecuted:Called to check if requested routine was executed.

Software Unit Interaction

In RoutineControl component the Interfaces are triggered by the Call backs from the DCM module when the service request is received. You can find more detailed description of each interaction below.

Start Ethernet Diag Routine

This interaction take place when start diag ethernet routine event(GW_Routine_0x068F_Start) is triggered from DCM callback.

Stop Ethernet Diag Routine

This interaction take place when stop diag ethernet routine event(GW_Routine_0x068F_Stop) is triggered from DCM callback.

Ethernet Diag RequestResult Extended

This interaction take place when ethernet diag routine request result event is triggered(GW_Routine_EthDiag_RequestResultExtended).

Ethernet Diag Request result Normal

This interaction take place when ethernet diag routine request result event is triggered(GW_Routine_EthDiag_RequestResultNormal).

Abort Routine

This interaction take place when ethernet diag routine abort event is triggered(GW_Routine_EthDiag_Abort).

Start Routine

This interaction take place when start event is triggered(GW_Routine_0x03E7_Start).

Stop Routine

This interaction take place when stop event is triggered(GW_Routine_0x03E7_Stop).

Request Result

This interaction take place when request routine result event is triggered(GW_Routine_0x03E7_RequestResult).

Abort Routine

This interaction take place when abort routine request event is triggered(GW_Routine_0x03E7_Abort).
