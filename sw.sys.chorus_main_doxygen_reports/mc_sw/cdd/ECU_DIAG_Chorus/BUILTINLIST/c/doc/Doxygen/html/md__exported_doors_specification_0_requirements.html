Obtention of the Node-Address from KN_ or NMH_ frames

The ICAS1 shall obtain the Node-Address from the received KN_ or NMH_ frames as per following:

for CAN or CANFD
The Node-Address are the first 10 bits (Byte order: Little-endian) from the "Identifier" for the KN_ and NMH_ frames
- PDU-ID
KN_ : 0x17F00 / Node-Address /
NM_ : 0x1B000 / Node-Address /


for VLAN
The Node-Address are the first 10 bits from the (Byte order: Little-endian) from the "Identifier" - started from bit 16th, for the KN_ and NMH_ frames

- PDU-ID
KN_ : 0x5 / Node-Address / 0C00
NM_ : 0x02 / Node-Address / 0C00


Example:
Instrument Cluster (Kombi in German)
Is connected to KCAN (0x12) and Komfort VLAN (0x64)

KCAN:
KN_Kombi –> Identifier = 0x17F00 014
NMH_Kombi –> Identifier = 0x1B00 014

VLAN:
KN_Kombi–> PDU-ID: 0x50140C00
NM_Kombi–> PDU-ID: 0x20140C00

[Hash function]

Whenever the ICAS1 applies the Hash function to a Node-Address "k", it shall perform the following calculation:

- k: Node-Address
- K: ICAS1 Node-Address = 0x010h
- m: Size of the Built-in Liste (Hash table) = 0x100h
- j: Recursion level (For Double Hashing)

H(j,k) ::= ( h1(k) + j*h2(k) ) mod m

h1(k) ::= ( ( m + k - K ) mod m )
h2(k) ::= ((k + 64) mod m/2 ) | 1

Remark:
In C language implementation of h2(k) extra pair of brackets should be added to ensure correct operator orders:
h2(k) ::= ((k + 64) mod (m/2) ) | 1
