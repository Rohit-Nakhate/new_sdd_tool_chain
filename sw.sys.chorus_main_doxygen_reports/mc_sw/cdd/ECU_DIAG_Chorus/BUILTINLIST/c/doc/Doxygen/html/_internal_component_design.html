Internal Component Design

The component Built In List consists of only the follwing SW unit: BuiltInList.c
The component Built In List provides following interfaces:

  • to DiagTask :
    • BIL_Init which is used to initialize BuiltInList. Which is called by DiagTask once during start of system.
    • BIL_Cyclic100ms - Called cyclically every 100ms to check if all the buses connected to GW are asleep.
  • to RoutingTask:
    • BIL_CyclicTimeoutCheck - Called cyclically every 25ms to check if ECUs on Built In List didn't timed out.
    • BIL_CyclicClamp15Poll - Called cyclically every 50ms to check if Clamp15 became ON. If Clamp15 is switched ON, Istverbau parameter of all ECUs on BuiltInList is set to zero.
  • BIL_NmhRcvCb & BIL_KnRcvCb that is called by ISOTpRouting, which is used to update the parameters when new message is received.
  • to DCM in order to:
    • read/write Sollverbau value via diagnosis adaptation channel DID 0x04A3 : BIL_Read_0x04A3, BIL_Write_0x04A3
    • read/write ContentList value via diagnosis adaptation channel DID 0x0502 : BIL_Read_0x0502, BIL_Write_0x0502
    • read Istverbau value via diagnosis adaptation channel DID 0x2A26 : BIL_Read_0x2A26
    • read SleepIndication value via diagnosis adaptation channel DID 0x2A27 : BIL_Read_0x2A27
    • read KDBit value via diagnosis adaptation channel DID 0x2A28 : BIL_Read_0x2A28
    • read DiagProt value via diagnosis adaptation channel DID 0x2A29 : BIL_Read_0x2A29
    • read BusID value via diagnosis adaptation channel DID 0x2A2D : BIL_Read_0x2A2D
    • read Authentizitat value via diagnosis adaptation channel DID 0x2EE1 : BIL_Read_0x2EE1
    • read NodeID value via diagnosis adaptation channel DID 0xF1B7 : BIL_Read_0xF1B7
    • read DiagProt value via diagnosis adaptation channel DID 0x2A29 : BIL_Read_0x2A29.
  • to DEM to set appropriate state for the DTC 0x300000, if
    • node send the message but the node is not configured in the sysytem.
    • index computation > 15, i.e; no free space is present in built in list.

Dynamic Behavior

Tasks

The component Built in List provides one interface, that is called cyclically by OSTask "DiagTask" (period = 100ms) : BIL_Cyclic100ms.
The component Built in List provides two interface, that is called cyclically by OSTask "RoutingTask" (period = 5ms) : BIL_CyclicTimeoutCheck, BIL_CyclicClamp15Poll.

see also Interaction Diagram "Cyclic behaviour" at the following location in Enterprise Architect:
