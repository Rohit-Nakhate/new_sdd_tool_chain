Internal Component Design

Internal Component Design

The component is splitted into the follwing SW units:

  • ErrHdl_Can_bus_Enable_condition - This is responsible for set/reset of enable condition of standard can and tDiagStartT
  • ErrHdl_Can_Bus_off_Detection - This is responsible for checking bus off condition and setting the related DTC.
  • ErrHdl_Can_Global_Timeout_Detection - This unit is responsible for monitoring Global timeout and than set/reset the corresponding DTC.
  • ErrHdl_Can_Single_Wire_Detection - This unit is responsible for monitoring Single wire error and than set/reset the corresponding DTC.
  • ErrHdl_Can_Vlan_Node_Timeouts_Detection - Responsible for checking if timeout has occured on each node and set the DTC. It also has an API to reset timeout when message is received.
  • ErrHdl_Can_Vlan_Nodes_Enable_Condition - Responsible for enable condition when MFL node is coded.
  • ErrHdl_Channel_Operation_Cycle - Cyclically checks for bus state change and handles operating cycles DTC of CAN, LIN nad remote CANS
  • ErrHdl_Dataset - Responsible to sets data set invalid DTCs at start of ICAS life cycle.
  • ErrHdl_DEM_Event_Map_Arrays - Provides node timemout DEM event ID.
  • ErrHdl_DEM_Interface - provides interface between Error handler and DEM component to set/reset DTC or enable condition.
  • ErrHdl_DK2_Can_Timeout_Detection - Responsible for DK2 node timeout DTCs and enable condition.
  • ErrHdl_DTC_State_To_DLI - Responsible to notify DLI Library about state of timeout DTCs.
  • ErrHdl_Ethernet_Communication - Responsible to inform that the eletrical fault occurred,Timeout occured,MAC address conflicts on ethernet bus.
  • ErrHdl_High_Temperature - Responsible to monitor high temperature errors.
  • ErrHdl_If - Provides an interface between BuiltInList and error handler. It is used to notify error handler that rebuild has taken place or new message is received, and than handling of DTC.
  • ErrHdl_Lin_Electrical_Failure_Detection - Handles setting/reseting LIN electrical failure DTC according to given Status
  • ErrHdl_Lin_Enable_Conditions - Responsible for setting LIN node, bus, or Lin global timeout enable condition.
  • ErrHdl_Lin_Frame_Errors_Detection - Cyclically monitors LIN node timeout and global timeout error, and set the corresponding error.
  • ErrHdl_Reset_Counter - Responsible for clearing/writing unexpected resets counter value to NVM when new data is available.
  • ErrHdl_Start_Diag_Timer - Responsible for counting Diag timer, KL15 timer, KL30 timer. and reset the timer if state changes from off to on.
  • ErrHdl_Steuergeraet_Defekt - Responsible to check if Steuergeraet Error DTC should be set to active and also responsible to reset flag which indicate that any of Steuergeraet DTC is currently handled.
  • ErrHdl_Voltage_Out_Of_Range_Error - Responsible to handle battery out of range errors and also to chose appropriate DTCs and DEM condition state based on VoltageRange.
  • ErrHdl_Vlan_Bus_Port_Enable_Condition - Cyclically monitors Vlan bus status. If bus state is active than set enable condition and reset the diag timer.
  • ErrHdl - Responsible for providing cyclic task for other error handler unit.

Software Unit Interfaces

There is interface between ErrHdl and other software unit in this component. ErrHdl provides cyclic task to other software unit.
The ErrHdl_Can_bus_Enable_provides an interface (ErrSetCanGlobalTimeoutDEMCond) to ErrHdl_if.c sets Can global timeout enable condition when at least one node is coded on bus.
ErrHdl_Can_Global_Timeout_Detection provides interface ErrIsCanGlobalTimeoutAllowed() to ErrHdl_Can_bus_Enable_condition to check if global timemout is allowed.
ErrHdl_Can_Bus_off_Detection provides interface (ErrIsBusOffCounterGreaterThanZero) to ErrHdl_Can_Global_Timeout_Detection and ErrHdl_Can_Vlan_Node_Timeouts_Detection to check if bussoff error has occured.
ErrHdl_Can_Vlan_Node_Timeouts_Detection provides an interface ErrResetNodeTimeoutCounterAndDTC to reset DTC and timer when new message is received used by ErrHdl_if, ErrGetNumOfCodedNodesOnBus to check the number of coded nodes and ErrGetBillIndexFromErrCodedNodesArr to get index of nodes, used by Global timeout detection and VLAN bus port enable condition.
ErrHdl_DEM_Event_Map_Arrays provides an interface (ErrGetNodeTimeoutDEMEventId) used to get DEM event ID of node timeout which is used by ErrHdl_Can_Vlan_Node_Timeouts_Detection.
ErrHdl_If provides interface (ErrGetBusState) to other Error handler to check bus state.
ErrHdl_DEM_Interface provides interface between Error handler and DEM component to set/reset DTC or enable condition.
ErrHdl_Slaves_Defect_Detection is used to monitor if DK2 built in list rebuild takes place. and also to handle DK2 DTC.
ErrHdl_Start_Diag_Timer provides interface

  • ErrTDiagStartTKL15HasElapsed to get the status of KL15 Diag timer, which is used by ErrHdl_Can_Global_Timeout_Detection and ErrHdl_Can_Vlan_Node_Timeouts_Detection.
  • ErrTDiagStartTKL30HasElapsed to get the status of KL30 Diag timer, which is used by ErrHdl_Can_Global_Timeout_Detection, ErrHdl_Can_Vlan_Node_Timeouts_Detection, ErrHdl_Can_bus_Enable_condition and ErrHdl_Lin_Enable_Conditions.
  • ErrResetTDiagStartT to reset Diag timer, which is used by ErrHdl_Can_bus_Enable_condition, ErrHdl_DEM_Interface, ErrHdl_Lin_Enable_Conditions

ErrHdl_Can_Bus_off_Detection provides (ErrSetResetCanBusOffCounter) external interface to BusMonitorConfig to check if Bus off failed has occured.
ErrHdl_If provides an external interface between Built in list component and Error handler component. It is used to get information when message in received, solverbau or node list is updated.
ErrHdl_Lin_Electrical_Failure_Detection provides a external interface to set DTC when LIN electrical failure is detected.
ErrHdl_Lin_Enable_Conditions provides an interface to GW_SollvSubbTeiln to set LIN enable condition.
ErrHdl_Reset_Counter provides a interface to DiagAppl_DCM_Service14 to update reset counter value when SID 14 is requested.

Dynamic Behavior

Tasks

The component Error Handler provides interface that is called cyclically by OSTask "ErrHandlerTask" (period = 10ms). The monitoring of error take place cyclically here. see also Interaction Diagram of Error handler cyclic at the following location in Enterprise Architect: VW_ICAS1.Chorus_6M.All_Cores.SoftwareDecomposition.ComplexDeviceDrivers.ErrHdl_MQB.SW Architecture.Dynamic

Interrupts

There is no interrupts directly handled by Error handler.

Events

there is 2 events in Error Handler ErrStartEv - During start of a event and ErrCyclicEv - cyclic event called every 10ms.

Operating Modes

Operation modes are used by DTCs and network diagnoses. Each bus has his own operation cycle.

  • New operation cycle is started when bus change state from inactive to active.
  • Operation cycle ends when bus change state from active to inactive.

Power Modes

The component need to work in the following power modes:

  • under voltage is below 5500.
  • Normal voltage between 5500+500 to 18500-500.
  • over volatge is above 18500.

As it need to set DTC during under voltage and over voltage.

Synchronization

Between Built In List and Error handler we have synchronization. When new coding of 0x04a3 or 0x0502 is requested handling of the timeouts is stopped. There is no interrupts synchronization in Error Handler.
