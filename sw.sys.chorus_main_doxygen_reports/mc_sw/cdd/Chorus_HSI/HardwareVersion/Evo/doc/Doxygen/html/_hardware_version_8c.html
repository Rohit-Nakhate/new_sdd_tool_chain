Detailed Description

Source file for HardwareVersion subcomponent of Chorus_HSI component.

Source
HardwareVersion.h
Revision
version
Author
gusanp
Date
12/12/2018


This module reads current HW version information from defined OTP flash area (set in .pdr/.vpdr). Furthermore, based on OTP data(HW version information) it also determinates the base variant, HW version number, and HW version type.
APIs are available for other SW modules to request the HW version and base variant in a readable format.

void HardwareVersion_GetHardwareVersion ( HardwareVersionType *  hwVersion)

Function that selects and stores the corresponding hardware version type based on hardware version information.

Based on the OTP data (hardware version information) read from OTP flash memory, this function creates an 32 bit integer by combining HW version IDs (in this order: HardwareVersionPlatform, HardwareVersionHighAscii and HardwareVersionLowAscii values) and based on this integer, it will select the corresponding hardware version type.

NOT_DEFINED_HW initializes the hardware version and then it is used to represent an unknown HW version type. It is important because in this case, the case should be later handled by FBL, so it does not start into APPL if no hardware version is given.
HardwareVersionCopy struct used to access HardwareVersionPlatform, HardwareVersionHighAscii and HardwareVersionLowAscii values and combining them, in this order, into a 32bit integer. Based on this integer value the corresponding HW version type is chosen.
VW_MEB19_EVO_B1_DEV, VW_MEB19_EVO_B1_MAX_DEV,VW_MEB19_EVO_B2_DEV, VW_MEB19_EVO_B2_TI_DEV, VW_MEB19_EVO_C1_DEV, VW_MEB19_EVO_C1_SERIAL, VW_MEB19_EVO_Cx_DEV, VW_MEB19_EVO_Cx_SERIAL represents HW version types.

void HardwareVersion_ReadOtp ( void  )

Function that reads the hardware version information from the OTP (one time programmable memory) flash memory, more exactly from diagnostic identifier 0xF1A3.

If the hardware version information wasn't read before, all the necessary informations will be read and stored into a struct.

Precondition
N/A
Postcondition
N/A
Parameters
void
Returns
void
Globals
HardwareVersionAlreadyReady [in]
HardwareVersionCopy [out]
HardwareVersionAlreadyReady represents if the hardware version was or wasn't read before from the OTP flash memory. If it is FALSE, the informations regarding hardware version will be read again and if it is TRUE, the information will not be read again and will be used from the stored local copy HardwareVersionCopy.
HardwareVersionCopy struct used to store all the information regarding hardware version from OTP flash memory.
