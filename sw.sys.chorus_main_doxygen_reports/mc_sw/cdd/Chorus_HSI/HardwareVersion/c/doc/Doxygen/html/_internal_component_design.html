Internal Component Design

Internal Component Design

The component has the following files that are developed by Continental:

  • HardwareVersion.c - This file implements the functionality of reading the current HW version and base variant from OTP flash memory area.
  • HardwareVersion.h - Header file for HardwareVersion.c. Contains the typedefs for HardwareVersion type, Base Variant type, Hardware Version information and Hardware Version number.
  • PostBuildHandler.c - This file implements the posibility for functions to select the required PB configuration during startup and according to ECU variant.
  • PostBuildHandler.h - Header file for PostBuildHandler.c.
Internal Component Design

Requirements Allocation

The existing requirements for HardwareVersion functionality are implemented at design level in both components - HardwareVersion (HardwareVersion.c) and PostBuildHandler (PostBuildHandler.c). As per above mentioned DOORS baseline (29.33), for this component there are no valid non functional requirements.

Software Unit Interfaces

See the header files:

  • HardwareVersion.h - Declaration file for reading OTP data and selecting the corresponding HardwareVersion type,Base Variant type, Hardware Version information and Hardware Version number. This information will be used by other functions.
  • PostBuildHandler.h - Declaration of function used for selecting the corresponding PB configuration. The functionality from HardwareVersion.c will be also be used here.

Interoperability is set to low because CDD is project specific.

Interaction

Interaction is set to medium because the component is used by other components to get information.

Criticality

Since HardwareVersion is non-safety and the interaction is Medium, the criticality is medium.

The technical complexity is medium because HardwareVersion is split in smaller units but has interactions with other components.

The component HardwareVersion is composed of only two unit and no complex alghoritms are implemented therefore the risk is low.

Testability

The component HardwareVersion requires other tools (like PLAST) besides SWATT for 100% tests coverage but all tests can be automated. So, the testability level is high.
