Warning
Requirement is not verified
The ICAS1 shall enable/disable the evaluation of the preconditions for all diagnostic requests depending on the State "OBD Driving Cycle" as follows:


- If this state is OBD_Driving_Cycle_notSet
then ICAS1 shall disable the evaluation of the preconditions.
This means that all diagnostic service requests will be accepted in terms of "preconditions"

- If this state is OBD_Driving_Cycle_setOnce
then ICAS1 shall check the defined preconditions, when ICAS1 receives a diagnostic request for a service which needs the evaluation of preconditions.
If at least one of the checks fails, then ICAS1 shall respond with NRC 0x22.

- If this state is OBD_Driving_Cycle_setOnce
then ICAS1 shall check the defined preconditions also while the function started by the service is active (refer to column "Applicable during active diagnostic service" in SWR_ICAS1_Diag_114). As soon as at least one precondition is getting false, ICAS1 shall return to the default function.
a) This applies to service 0x28: When the preconditions are no more fulfilled, mute mode shall be aborted as described in SWR_ICAS1_Diag_211.
b) This applies to service 0x85: When the preconditions are no more fulfilled, DTC setting shall be reenabled. This is not described in SWR_ICAS1_Diag_341 !



The state "OBD Driving Cycle" is determined according to the state chart in SWR_ICAS1_Diag_121.
