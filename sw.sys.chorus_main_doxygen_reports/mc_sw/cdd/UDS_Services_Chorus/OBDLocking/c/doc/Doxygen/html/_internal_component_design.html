Internal Component Design

Internal Component Design

The component OBD Locking consists of only the follwing SW unit: OBDLocking

The component OBDLocking provides interface to cyclically monitors ORU status and check if routine 300 is called. If online remote update is taking place than OBDLocking status is set to active. If routine 300 is called than deactivate OBD Locking status.
It provides one interface OBD_locking_update - This interface is cyclically called by OSTask "DiagTask" every 10ms. Which is used to check if ORU is Active/Deactive. And also to check for routine 300 status.
Additionally Get_OBD_locking_active function is called by ConfigDiag to check for OBD status.

OBDLocking consist of only one unit OBDLocking.c. All OBDLocking requirement is satisfied by this unit.
