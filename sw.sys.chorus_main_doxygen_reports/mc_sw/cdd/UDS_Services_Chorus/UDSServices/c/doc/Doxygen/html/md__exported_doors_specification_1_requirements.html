RoutineInfo
The ICAS1 shall send the RoutineInfo parameter (1Byte) containing following parameters in the Positive Response to a RequestRoutineResults request

Remark:

- Bits 0 to 3: RoutineStatus
- 0: RoutineInactive –> If the routine has not been executed in the current operation cycle.
- 1: RoutineActive –> If the routine is being executed.
- 2: RoutineAborted –> If the Routine was stopped because of
- an error
- a safety reason
- a session change to default session.
- a KL15 change to KL15 OFF
- 3: RoutineExecutionStuck –> Routine is being executed, but cannot be stopped.
- 4: RoutineFinishedCorrectly –> the routine has been successfully executed in the current operation cycle
- 5 :RoutineReturnedToEcuControl –> the routine has been successfully stopped via StopRoutine request.
- 6-F: RoutineSystemSupplierDefinedReachedStatus –> Reserved Status for specific purposes. If a different status is needed for a routine, this will be defined in the corresponding requirement for the routine.

- Bits 4 to 7: RoutineResult This parameter is used to describe if the routine has generated results, and if they are correct or not.
- 0: NoResultsAvailable
- 1: IncorrectResults
- 2: CorrectResults

NRC 0x22 ConditionsNotCorrect

The ICAS1 shall respond to a 0x31 request with this NRC in following situations:
- when the ICAS1 receives a RoutineControl request, but at least one of the preconditions is not fulfilled
- when the ICAS1 executes a Routine, but at least one of the conditions stop to be fulfilled
- when the ICAS1 receives a StopRoutine request, which can or must not be stoped during its execution. This case is to be defined in the corresponding routine.

Service Description

This service is cyclically requested by a diagnostic tester which previously requested a non-default session. The ICAS1 uses the reception of these requests to implement the Keep-Alive-Logic

Whenever a non-default session is active, the ICAS1 shall reset the S3server timer everytime it receives a TesterPresent request. If the TesterPresent request is not received for a time longer than S3server timeout, then the ICAS1 will perform a transition to default session as specified in transition 11 (in Application mode) in [Session transitions in Application mode]

This service shall work also with KL15 OFF

Session: 0x01 Default Session
0x02 ECU Programming-Session
0x03 Extended-Diagnostic-Session
0x40 VW-EOLSession
0x4F Development-Session

Remark:
In Bootloader mode, the S3server timeout leads to a reset, as described in [Session transitions in Application mode]
