Software Unit Interfaces

The UDSServices is composed from thirteen SW units,
The DiagAppl_DEM_ExtendedData.c file mainly provides the interfaces to read Mileage, TimeStamp, TimeIndication & Steuergerat_Defekt with the RTE external interface calls.
The DiagAppl_Impl.c file handles the preconditions for the usage of services
The DiagAppl_DCM_ServiceUnsupported.c handles the unsupported services
The DiagAppl_DCM_Service19.c file handles read the DTCs extended data with the help of service0x19.
The DiagAppl_DCM_Service10.c file handles change between diffrent diagnostic sessions with help of service0x10.
The DiagAppl_DCM_Service11.c file handles ECU soft reset with the help of service0x11.
The DiagAppl_DCM_Service85.c file handles Control DTC setting with the help of service0x85.
The DiagAppl_DCM_Service14.c file handles interfaces to clear diagnostic information for the DTCs with the help of service0x14.
The DiagAppl_DCM_Service28.c file handles control the communication by enable/disable of Tx&Rx messages.
The DiagAppl_DCM_Service87.c file handles to set/get the new baudrates for CAN channel.

Power Modes

There is no special power mode used for UDSServices.
The Chorus supply voltage for UDSServices should not be less than 10volts and not be more than 17volts, is the precondition for diagnostics.
