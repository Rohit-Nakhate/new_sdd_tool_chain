◆ LinChannelHwStatus

TLinStatus LinChannelHwStatus[LIN_CHANNELS_COUNT]
static

This array will store the hardware status for the LIN channels

For each LIN channel the status will be read using the l_ifc_read_status(ifc) interface provided by LIN FlexDriver component The status for a specific LIN channel will be updated inside the LinInt_ReadHwStatus()
function that will provide the requested LIN interface as parameter. For reading the HW
status of a LIN channel, the LinInt_GetHwStatus() function will be used.

Remarks
range The size of this array will correspond to the number of used LIN channels
values The l_ifc_read_status will return a 16-bit status frame
context {LinTask/C1, LinInt_ReadHwStatus, "Write data" }
context {LinTask/C1, LinInt_GetHwStatus, "Read data"}
protection {none, "Data is written only from LinTask"}
storage class {init ram}

◆ LinChannelModeRequests

TLinChannelModeRequest LinChannelModeRequests[LIN_CHANNELS_COUNT]
static

This variable will store the requests for switching the state of LIN channels.

The request to switch the state of the LIN channels from other components will be handled by external interfaces described in the SWA. Inside the Lin Integration component we will use this global variable to keep track and cyclically handle the mode change requests for all the LINs.

Remarks
range The size of this array will correspond to the number of used LIN channels.
values TRUE or FALSE.
unit state of the request - TRUE = requested, FALSE = not requested.
context {LinTask/C1, LinInt_HandleModeChanges_Cyclic, "the flags are read and reset if set" }
context {SystemControl/C2, LinInt_ComChannel_RequestOnlineMode, "onlineModeRequested flag is set "}
context {SystemControl/C2, LinInt_ComChannel_RequestOfflineMode, "offlineModeRequested flag is set"}
context {SystemControl/C2, LinInt_ComChannel_RequestNormalMode, "normalModeRequested flag is set"}
context {SystemControl/C2, LinInt_ComChannel_RequestSleepMode, "sleepModeRequested flag is set" }
protection {none, "Setting is done on Core2 and reseting on Core1 but the access is atomic (the cache is disabled)"}
storage class {init ram}
