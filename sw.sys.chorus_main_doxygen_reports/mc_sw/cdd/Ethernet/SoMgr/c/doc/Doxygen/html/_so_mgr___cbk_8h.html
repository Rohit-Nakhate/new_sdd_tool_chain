uint16  Length 
)

The TCP/IP stack calls this function after the data has been acknowledged by the peer for TCP or was sent to the lower layer driver using UDP.

Precondition
None
Postcondition
None
Parameters
[in]SocketId- the socket on which the requests is received
[in]Length- length of the data pointed out by pBuff
Returns
None
Remarks
TcpIp_TcpTransmit can return E_NOT_OK if request has not been accepted, e.g. due to a lack of buffer space or the socket is not connected. If socket isn't online then probably was closed (or in transition to) and resources will be freed. Otherwise, in case when not enough buffer is available in TCP/IP, it was decided not to free here the pTxWin, this action being left in the decision of the next socket transmission (SoMgr_Main_TcpTransmit)
