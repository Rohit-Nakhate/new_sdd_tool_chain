Std_ReturnType ConfigLinDiagRead ( Dcm_OpStatusType  OpStatus,
uint8 *  Data 
)

Copies data from ConfigLinDiag_Read_Write_DID_buf to the DCM buffer after successful collecting of a slave response.

Precondition
Function is expected to be called after successfull execution of LIN DK1/DK2 request, when the ConfigLinDiag_Read_Write_DID_buf is filled with response data and #responseLengthDCM with response data length.
Postcondition
Terminates processing of the DCM request.
Parameters
[in]OpStatus- Status of the DCM request operation
[out]Data- DCM buffer for Read Data By Identifier response data filled with data received from the LIN slave
Returns
DCM_E_OK in case of valid parameter values
DCM_E_NORESPONSEFROMSUBNETCOMPONENT in case of invalid slave response length
DCM_E_NOT_OK in case of invalid paramater values
Globals
ConfigLinDiag_Read_Write_DID_buf - read in order to copy data to the DCM buffer

Test Method

◆ ConfigLinDiagReadWrite()

Std_ReturnType ConfigLinDiagReadWrite ( const tLinDiagReq *  LinDiagReq)

Handles LIN DK1/DK2 request processing.

This function is expected to be called cyclically in order to transmit a request and collect a response.

Precondition
No preconditions.
Postcondition
LinDiag request processing triggered.
Parameters
[in]RequestedDID- RecordDataIdentifier of the single service 22/2E diagnose request to be executed
[in,out]Data- Pointer to request/response buffer
[in]RequestedSID- requested SID
[out]Dcm_NegativeResponseCodeType- negative response code for DCM
[in]LinSlaveID- ID of asked LIN slave
Returns
DCM_E_OK, if LIN diag request is successful
DCM_E_PENDING, if LIN diag request in pending
DCM_E_NOT_OK, if pointer are invalid or for received NRC from LIN
Globals
#u8_LinCodingActive - variable written with coding triggering status
#responseLengthDCM - update with slave response length in case of request processing finished
#u8_LinDiagRequestStarted - read in order to check if a LIN DK1/DK2 identification request is currently beeing processed
