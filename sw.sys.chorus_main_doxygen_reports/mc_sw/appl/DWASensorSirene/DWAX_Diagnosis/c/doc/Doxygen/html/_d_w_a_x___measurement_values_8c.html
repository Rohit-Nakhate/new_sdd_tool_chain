Std_ReturnType DWAX_Read_Measure_Values_0x198B ( Dcm_OpStatusType  OpStatus,
uint8 *  Data 
)

This function handles the read request 0x22 0x198B.

Function that reads the measure values of DID 0x198B (responsible for status sirene).
If some of the parameter signals of DID are active, then the corresponding bit of DID will be set to active.

Precondition
N/A
Postcondition
N/A
Parameters
[in]OpStatus- Operation status
range Dcm_OpStatusType
resolution N/A
unit N/A
[out]*Data- Buffer for copying response data
range uint8
resolution N/A
unit N/A
Returns
Std_ReturnType retVal - Value error code for DCM, standard return AUTOSAR type.
range 0 - E_OK | 1 - E_NOT_OK, defined in AUTOSAR Standard.
resolution N/A
unit N/A
Globals
E_OK [in] - If all th signal parameters were succesfully read.
MESSAGE_0x198B_ARRAY_SIZE [in] - Array size of message 0x198D
IpduStatus_Sounders_01 [in] - Used to calculate the sensor timeout.
Rte_StatusMask_TimeoutError [in] - Used to calculate the sensor timeout.
E_NOT_OK [in] - if any of the signal parameters couldn't be read properly or if Data is empty.
RTE_E_OK [in] - Checks if the read of signals was succesful
Input->Output-Correlation
E_OK indicates that all signal parameters were read succesfully.
MESSAGE_0x198B_ARRAY_SIZE indicates the size of the array in which the value of signal parameters will be stored (auTempArray).
IpduStatus_Sounders_01 and Rte_StatusMask_TimeoutError used to calculate if the sensor timeout is true or false (if FALSE, there is communication and if TRUE, there isn't).
E_NOT_OK indicates that at least one of the signal parameters couldn't be read properly.
RTE_E_OK indicates that the read of signals was succesful.
