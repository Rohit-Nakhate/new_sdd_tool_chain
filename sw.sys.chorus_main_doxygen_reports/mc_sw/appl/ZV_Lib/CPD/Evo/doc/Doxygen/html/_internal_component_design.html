Internal Component Design

Internal Component Design

CPD is part of the ZV_Lib composition and contains manually developed files. The following files are developed by Conti and are required for the proper integration of the library:

  • CPD_Task.c - Code is called by scheduler which will init, start, run and stop the operation of CPD
  • CPD_Task.h - Header file for CPD_Task.c
  • CPD_NvRam.h - Header file needed to connect memory and lib
  • CPD_MemMap.h - Header file used during RTE generation

Operating Modes

Not applicable because CPD component runs in high-power mode. At the moment, CPD doesn't require a special power modes, it only integrates library functions.

Power Modes

As mentioned in above chapter, the current functionality is available only in high-power mode.

Exception and Error Handling

So far, the CPD functions do not handle errors via return value, they are only void type. There are also no DTC or DEM events.

Synchronization

Since the component is designed to run on a single core, the concurrent access of the component is not required.

Interoperability

Interoperability is set to low because CPD depends on other components.

Interaction

Interaction is set to low because the component has a reduced number of non RTE dependencies. See VW_ICAS1.Chorus.All_Cores.SoftwareDecomposition.ApplicationLayer.ZV_Lib.SW Architecture.CPD.CPD Static.CPD Dependecies.

Criticality

Criticality is set to low because CPD is a QM component, the level of interaction is low. Also, this component has been recently integrated.

Technical complexity

Technical complexity is set to low because so far CPD doesn't implement any algorithm only integrates the library functions and interaction component is low.

Risks

Risks is set to low because CPD has only one SW unit. See VW_ICAS1.Chorus.All_Cores.SoftwareDecomposition.ApplicationLayer.ZV_Lib.SW Architecture.CPD.CPD Dynamic.CPD Init and Cyclic.

Testability

Testability is set to low because CPD can be tested on SWATT and basic tests should be performed on the target.
