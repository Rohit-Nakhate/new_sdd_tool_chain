Helper function that returns the communication status based on the state of the bus, the pdu status and the timeout timer value

This function sets the communication error status based on the requirements.
While bus sleep, the communication status should be set to init value.
If bus is active, if PDU status is NoError, the communication status is set to NoError, else if the timeout is not elapsed, we set it to init (bus is not fully awake), otherwise, the communication status is set to error.

Function the returns the communication error status for a specific PDU

This function cyclically updates the pdu counters accordingly to the communication error status requirements.
After each BUS WAKE-UP, the counters are reset. They are decrementing only if the BUS WAKE-UP is not finished (after the timeout has elapsed or when we receive a NoError pdu status).
