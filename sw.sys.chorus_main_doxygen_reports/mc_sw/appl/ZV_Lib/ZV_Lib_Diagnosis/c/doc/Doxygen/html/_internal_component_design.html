Internal Component Design

The component ZV_Lib_Diagnosis has five units: CPD_ADAPTATION, ZV_ADAPTATION, ZvDiagControl, ZV_Routines, ZV_MeasurementValues.

  • CPD_ADAPTATION.c - source file for CPD Adaptation
  • CPD_ADAPTATION.h - header file for CPD Adaptation
  • ZV_ADAPTATION.c - source file for ZV Adaptation
  • ZV_ADAPTATION.h - header file for ZV Adaptation
  • ZvDiagControl.c - Zv diagnostic wrapper to check if diagnostics services are allowed.
  • ZvDiagControl.h - header file for ZvDiagControl
  • ZV_Routines.c - file contains implementation of routine control
  • ZV_Routines.h - header file for ZV_Routines
  • ZV_MeasurementValues.c - source file for ZV Measurement Values
  • ZV_MeasurementValues.h - header file for ZV Measurement Values

Software Unit Interfaces

Adaptation is composed from two units: CPD_ADAPTATION, ZV_ADAPTATION. Both are referred as SWC_ADAPTATION.
Both of them handle the implementation of the read-write operations for certain DIDs.
DiagControl has one unit: ZvDiagControl.
The unit ZvDiagControl check if diagnostics services are allowed.
RoutineControl has one unit: ZV_Routines.
The unit ZV_Routines handles the implementation of abort routines.
MeasurementValues has one unit: ZV_MeasurementValues.
The unit ZV_MeasurementValues handles the measurement configuration parameters written via Diagnosis and it is triggered by DCM in order to read requested data.

Power Modes

The current functionality is available only in high-power mode.

Exception and Error Handling

The negative response codes are passed to the DCM using the following output parameter: ErrorCode.
If the vehicle is in externally locked state then the trigger write operation for the DIDs will return DCM_E_NOT_OK and ErrorCode DCM_E_GENERALREJECT.
If the diagnostic session or payload is not according to DCM configuration based on the requirements then the diagnostic READ/WRITE request will be rejected with ErrorCode GENERAL_REJECT or GENERAL_PROGRAMMING_FAILURE.

Synchronization

Since the component is designed to run on a single core, the request variables are protected by means of critical sections Suspend/ResumeAllInterrupts.
