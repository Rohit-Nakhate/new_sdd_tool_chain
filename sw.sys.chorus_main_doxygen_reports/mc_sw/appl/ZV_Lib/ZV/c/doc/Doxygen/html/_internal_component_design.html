Internal Component Design

Internal Component Design

The component is splitted into the follwing SW units:

  • ZV_Task.c - Will init, stop and run cyclicly the operation of ZV Code when called by Scheduler. It is also needed for extending CAN frames duration.
  • ZV_Task.h - Header file for ZV_Task.c.
  • ZV_NvRam.h - Header file is needed for connecting memory and lib.
  • ZV_ExtraDefinitions.h - Header file is needed to override functions by RTE_Light generator.
  • ZV_MemMap.h - Header file used during RTE generation.

Software Unit Interfaces

The entire functionality of ZV is contained by a single unit (ZV_Task) and therefore no interfaces between units are required.

Software Unit Interaction

ZV has no interactions between software units.

Dynamic Behavior

ZV has no dynamic behavior between SW units.

No OSEK Tasks needs to be configured by the module.

Interrupts

No interrupt needs to be configured for by the module.

Events

No OSEK event needs to be configured for by the module.

Operating Modes

ZV operates only on high-power mode. Sincer ZV is only a wrapper the 3rd party lib it doesn't require special power modes.

Power Modes

ZV runs only in high power since it doesn't request special power modes, as mentioned above.

Exception and Error Handling

ZV does not handle errors via return value or a return param. There are also no DTC or DEM events.

Synchronization

Since ZV is designed to run on a single core, concurent access of the component is not required.

Interoperability

The component ZV depends on other components. A wrapper could be developed for ZV but it would be to expensive.

Interaction

Interaction is set to low because the component has a reduced number of non RTE dependencies.

Criticality

Since ZV is non-safety, the interaction is Low and the component was not integrated recently, the criticality is Medium.

Technical complexity

The technical complexity of ZV is Low since the interaction is on Low.

Risks

The component ZV is composed of only one unit and no complex alghoritms are implemented therefore the risk is Low.

Testability

The component ZV requires other tools (like PLAST) besides SWAT for 100% tests coverage but all tests can be automated.
